# Utility trong typescript
-> Dùng type khi cần định nghĩa cấu trúc 1 objects
Dùng interface và class khi cần hướng đối tượng implement, extends lẫn nhau.

-> type A = ReturnType<typeof f1>; => type A là giá trị trả về của hàm f1

-> type C = keyof o2; => type C có giá trị là 1 trong các key của object o2

-> const p: Promise<string> = new Promise(resolve => { resolve("Hello"); });

-> type E = Awaited<ReturnType<typeof fPromise>> => giá trị trả về của hàm fPromise sau khi await thành kiểu gì

-> type F = "a" | "b" | "c"; type B = "a"; type Result = Exclude<A, B>; => "b" | "c" => loại bỏ giá trị nào khỏi tập giá trị Union

-> type G = Omit<User, "password">; => loại bỏ key khỏi object type

-> type D = Pick<o3, "error" | "a">; => type D giống object o3 nhưng chỉ tính có 2 fields "error" và "a"

-> type B = typeof o1; => type B tự suy ra từ object o1, o1 phải là 1 object instance nhé

-> type H = InstanceType<typeof ClassName> => lấy ra instance type của class, phải dùng typeof với 1 class name
typeof với class name, function các thứ thì là kiểu () => new User(), phải đặt trong <>
typeof với object instance thì trả về trực tiếp kiểu của object luôn

-> type MyAwaited<T> = T extends Promise<infer U> ? U : T;
type A = MyAwaited<Promise<string>>; // string
infer chỉ dùng trong conditional, nó lấy type bên trong của 1 cái gì

# File declaration.d.ts

1 file.ts export sẽ coi là 1 module và 1 file.ts khác luôn có thể import để dùng như JS => luôn dùng
Nếu 1 file.ts k có export thì file khác có thể thêm /// <reference path="file gốc.ts" /> thì dùng được code file gốc như bth => k nên dùng
file.d.ts là file global declaration chỉ định nghĩa type global, k có code thực thi. File này đặc biệt vì đặt ở bất cứ đâu, k cần export mà các file khác vẫn dùng bth.



Thêm vào tsconfig.json để nhận diện file chỉ khi k nằm trong thư mục chứa các file ts và k nằm trong thư mục được tsconfig.json quét thì thêm: {
  "include": ["src", "declaration.d.ts"]
}

- Trong file này chỉ được khai báo, k được có implementation, k thể gán =, có thể mở rộng kiểu dữ liệu có sẵn or khai báo kiểu mới. Bién khai báo trong file này là toàn cục
- Nếu dùng declare sẽ không tạo ra code JavaScript khi biên dịch, tức chỉ khai báo dùng trong TS chứ kp code thực tế. 
- Khai báo thì dùng global trực tiếp mà k cần import, declare cũng vậy. Còn nếu export sẽ tự hiểu là dùng riêng trong các module thì phải import, nó sẽ k còn toàn cục.
=> Chốt: thường dùng khai báo biến global, hiếm khi dùng declare.

Vd biến export ra k còn là toàn cục:
declare const API_URL: string;
declare function fetchData(url: string): Promise<any>;
export { API_URL, fetchData };
VD khi dùng typescript react mà bundler không hỗ trợ file svg. Ta phải có file.d.ts declare module "*.svg" mới chạy được, hoặc tải plugin hỗ trợ.
VD: 
declare const TEST_CONST: string; => Dùng như này thì khai báo giá trị ở đâu? K thể gán vì nó sẽ tưởng là redeclare lại biến const là lỗi. Thường chỉ dùng v khi có file js có biến sẵn rồi và khai báo type trong typescript để dùng file js trong ts thôi.

VD bth dùng try catch mà bắt từng loại error thì hay kiểu:
if(exception is UnauthorizedError) ....
else if (exception is RandomError) ....
Nhưng nếu làm v, hàm try throw 1 error mới ra thêm 1 đk check thì từng chỗ tái sử dụng hàm đó mà catch Error như v sẽ phải sửa và ta k thể kiểm soát hết được
Giải pháp trong typescript là biến error type thành phạm vi 1 list const, thì nếu list thay đổi, trình biên dịch sẽ báo lỗi typescript ngay
VD ban đầu:
function err<E extends { reason: string }>(error: E) { ... }
Biến reason thành const string:
function err<const R extends string, E extends { reason: R }>(error: E) { ... }
Như vậy typescript tự động detect hàm gọi err trả ra những const nào báo lỗi mọi nơi nếu thiếu nó

# Generic và infer
-> extends == là subset của
string extends string | number => true
string | number extends string => false
T extends string với T = string | number => bị distribute ra (true | false)
[T] extends [string] => thì lại true

Dùng infer khi cần lấy type bên trong của conditional ra.
VD: type MyAwaited<T> = T extends Promise<infer U> ? U : T; type A = MyAwaited<Promise<string>>; // string
VD: type Generic<T> = T extends Array<infer U> ? U extends string ? number : U : never; const b = ["abc", 1]; const A: Generic<typeof b> // string | number
VD: type CustomReturnType<T extends (...args: any[]) => any> = T extends (...args: any[]) => infer R ? R : never; => chính là ReturnType
VD: type CustomAwait<T> = T extends Promise<infer R> ? R : never; => hoặc viết rõ CustomAwait<T extends Promise<any>> cũng được 

Advance infer:
VD có: "key:value" cần lấy ra object có type { key, value }
type KeyValueSplitter<T extends string> = T extends `{infer K}:{infer V}` ? { key: K, value: V } : never;
type A = KeyValueSplitter<"name:hieu">; => A là const type { key: name, value: hieu }

-> Loop over key of object type
& với type thì chính là hợp
type O = {
  readonly name: string
  age?: number,
  comment: string
}
VD1: type New<T> = { [P in keyof T]: P }
type A: New<O> = { name: "name", age: "age", comment: "comment" } // value là fix cứng string
VD2: type New<T> = { [P in keyof T]: T[P] extends string ? number : T[P] } // copy kiểu của T nhưng nếu string sẽ đổi thành type number
VD3: type New<T> = {
  -readonly [P in keyof T]-?: T[P] => thêm readonly mọi trường, k có "-" sẽ là thêm readonly mọi trường, tương tự xoá optional mọi trường
}
VD4: 
type User = {
  readonly id: number
  readonly name: string
  readonly age: number
}
type MutableField<T, K extends keyof T> =
  Omit<T, K> & {
    -readonly [P in K]: T[P]
  }
type UserEditableName = MutableField<User, "name" | "age"> => chỉ bỏ readonly khỏi trường "name" và "age". Cơ chế là omit trường đó rồi thêm lại với &
VD5: 
type New<T extends Record<string, any>> = {
  [P in keyof T as `get${Capitalize<P & string>}`]: () => T[P]
}
// [P in keyof T] => để duyệt foreach từng key, nếu tiếp tục là 1 object thì P chỉ ở vế phải, là value chứ k được dùng P tạo key mới
// as để rename field
// Do muốn dùng Capitalize thì P phải là string mà nó k đủ thông minh để biết P là string thì cứ thêm & string để hợp thêm type string vào là được thôi. Chú ý là & chứ kp as vì as chỉ rename thôi
const a: New<O> = {
  getName: () => "abc",
  getAge: () => 12
}

# File declaration.d.ts
1 file.ts export coi là 1 module và 1 file.ts khác luôn có thể import để dùng như JS => luôn dùng
Nếu 1 file.ts k có export thì file khác có thể thêm /// <reference path="file gốc.ts" /> thì dùng được code file gốc như bth => bỏ
file.d.ts là file global declaration chỉ định nghĩa type global. Nó đặc biệt vì k cần export hay reference vẫn dùng đc ở các file khác.

- Nếu file k nằm trong thư mục tsconfig.json quét thì phải thêm: { "include": ["src", "declaration.d.ts"] }
- File này chỉ được khai báo kiểu mới or mở rộng kiểu dữ liệu có sẵn, k được có code logic, k thể gán -
- Dùng keyword declare sẽ không tạo ra code JS khi biên dịch, tức chỉ khai báo dùng trong TS chứ kp code thực tế. 
- Bién khai báo trong file luôn là toàn cục. Nếu dùng export thì mất tính năng toàn cục và chạy như 1 module bth.
Vd biến export ra k còn là toàn cục:
declare const API_URL: string;
declare function fetchData(url: string): Promise<any>;
export { API_URL, fetchData };
VD khi dùng typescript react mà bundler không hỗ trợ file svg thì phải có file.d.ts declare module "*.svg" mới chạy được, hoặc tải plugin hỗ trợ.
VD: declare const TEST_CONST: string; => Nếu dùng như này thì khai báo giá trị ở đâu? k thể gán vì nó sẽ tưởng là redeclare lại biến const là lỗi. Thường chỉ dùng v khi có file js có biến sẵn rồi và khai báo type trong typescript để dùng file js trong ts thôi.



# Keywords satisfies
-> UC1:
var Obj = {
  value: string | number
};
const option = {
  value: "abc"
};
var x = option.value.toLowerCase();
Muốn option theo kiểu Obj để khi thêm thừa các trường rác khác, nó sẽ báo lỗi thì phải const option: Obj ...
Nhưng dùng vậy thì option.value.toLowerCase() sẽ báo lỗi vì nó tưởng là string | number chứ kp string

Fix đảm bảo option thỏa mãn Obj nhưng vẫn giữ type của riêng nó:
const option = {
  value: "abc",
  mode: "simple"
} satisfies Obj;
=> dùng sau khi gán 1 trong các giá trị của | 
=> luôn đảm bảo biến có type gì đó hoặc bind với type gì đó nếu dùng type quá 1 lần. Nếu chỉ dùng 1 lần thì k cần định nghĩa type riêng

-> UC2: 
type RGB = `#{string}` | [number,number,number];
type PrimaryColor = "red" | "green" | "blue";
const colors = {
  blur: [0,0,255],
  red: "#F00",
  blue: #0F0",
}
console.log(colors.red.toLowerCase());
Muốn colors phải có đúng type để k có thừa trường, hoặc dùng type này ở đâu đó khác. Nếu dùng colors: Record<PrimaryColor, RGB> đảm bảo đúng type nhưng toLowerCase() báo lỗi
Fix lấy 1 trong các giá trị của | thì:
const colors = {
  blur: [0,0,255],
  red: "#F00",
  blue: #0F0",
} satisfies Record<PrimaryColor, RGB>;

-> UC3:
type Grade = "A" | "B" | "C" | "D" | "F" | "E";
function getGrade(grade: Grade): string {
  switch (grade) {
    case "A":
      return "Excellent";
    case "B":
      return "Good";
    case "C":
      return "Average";
    case "D":
      return "Below Average";
    case "F":
      return "Failing";
    default:
      throw new Error(`Invalid grade:: ${grade satisfies never}`);
  }
}
=> satisfies never đảm bảo 1 biến không thể là các type khác khi rơi vào case này, khiến cho các type khác phải định nghĩ đủ trong switch case.



# Generic type
Khi có nhiều type k xác định nhưng cùng common gì đó.
VD tạo ra generic type Data cho ApiResponse:
type ApiResponse<Data extends object = { status: number }> = {
  data: Data
  isError: boolean
}
Type Data là generic mà k xác định sẽ default là { status: number }. Nó cũng phải là type object, VD k được dùng ApiResponse<string>
type UserResponse = ApiResponse<{ name: string, age: number }>;
const userResponse: UserResponse = { data: { name: "a", age: 1 }, isError: false };
const userStatus: ApiResponse = { data: { status: 10 }, isError: false };

-> Biến thành const:
- Nếu 1 object dùng as const sẽ biến các giá trị bên trong thành fix.
